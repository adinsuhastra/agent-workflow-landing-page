# Workflow Notes

## Task Plan

The task was to create a small local portfolio project for the CO/AI Vibe Coder application path.

Project constraints:
- Static HTML, CSS, and JavaScript only.
- No paid tools.
- No external API keys.
- No fake client claims.
- No GitHub publication yet.
- No application email or submission.

## How Codex Was Used

Codex helped with:
- Turning the brief into a small frontend structure.
- Drafting the HTML sections.
- Writing responsive CSS.
- Adding small JavaScript interactions.
- Creating README and workflow notes.
- Preparing a build report.

Codex did not:
- Publish to GitHub.
- Send an email.
- Submit an application.
- Claim this was client work.
- Add credentials, payment flows, or external services.

## Human Review Approach

The project is designed to be checked manually before publication.

Review checklist:
- Confirm all text presents the page as a self-made demo.
- Confirm the page opens locally.
- Confirm card filtering works.
- Confirm checklist progress updates.
- Confirm the copy prompt button responds.
- Confirm the layout remains readable on smaller screens.
- Confirm README and notes accurately describe the project.

## Risks Avoided

- No fake professional or client claims.
- No paid tools or platform dependency.
- No private data.
- No external API keys.
- No hidden tracking.
- No complex framework setup that could slow review.
- No public publishing before user approval.
